# 스킬 생성기

Claude의 기능을 전문 지식, 워크플로우, 도구와 함께 확장하는 효과적인 스킬을 만들기 위한 가이드를 제공합니다.

*한글화 버전 - 2025.12.01 업데이트*

## 스킬에 대하여

스킬은 전문 지식, 워크플로우, 도구를 제공하여 Claude의 기능을 확장하는 모듈식, 독립적인 패키지입니다. Claude를 범용 에이전트에서 절차적 지식을 갖춘 전문 에이전트로 변환시킵니다.

### 스킬이 제공하는 것

1. 전문 워크플로우 - 특정 도메인을 위한 다단계 절차
2. 도구 통합 - 특정 파일 형식이나 API 작업을 위한 지침
3. 도메인 전문성 - 회사별 지식, 스키마, 비즈니스 로직
4. 번들 리소스 - 복잡하고 반복적인 작업을 위한 스크립트, 참조 자료, 자산

## 핵심 원칙

### 간결함이 핵심

컨텍스트 창은 공공재입니다. 스킬은 Claude가 필요로 하는 모든 것과 컨텍스트 창을 공유합니다: 시스템 프롬프트, 대화 기록, 다른 스킬의 메타데이터, 실제 사용자 요청.

**기본 가정: Claude는 이미 매우 똑똑합니다.** Claude가 이미 가지고 있지 않은 컨텍스트만 추가하세요. 각 정보에 대해 질문하세요: "Claude가 정말 이 설명이 필요한가?" 그리고 "이 단락이 토큰 비용을 정당화하는가?"

장황한 설명보다는 간결한 예제를 선호하세요.

### 적절한 자유도 설정

작업의 취약성과 가변성에 맞춰 구체성 수준을 조정하세요:

**높은 자유도 (텍스트 기반 지침)**: 여러 접근 방식이 유효하거나, 결정이 상황에 의존하거나, 경험적 방법이 접근을 안내할 때 사용.

**중간 자유도 (매개변수가 있는 의사코드나 스크립트)**: 선호되는 패턴이 존재하거나, 약간의 변형이 허용되거나, 구성이 동작에 영향을 줄 때 사용.

**낮은 자유도 (구체적인 스크립트, 적은 매개변수)**: 작업이 취약하고 오류가 발생하기 쉽거나, 일관성이 중요하거나, 특정 순서가 따라져야 할 때 사용.

## 스킬 구조

```
스킬명/
├── SKILL.md (필수)
│   ├── YAML frontmatter 메타데이터 (필수)
│   │   ├── name: (필수)
│   │   └── description: (필수)
│   └── 마크다운 지침 (필수)
└── 번들 리소스 (선택사항)
    ├── scripts/          - 실행 가능한 코드 (Python/Bash 등)
    ├── references/       - 필요에 따라 컨텍스트에 로드될 문서
    └── assets/           - 출력에 사용되는 파일들 (템플릿, 아이콘, 폰트 등)
```

### SKILL.md (필수)

- **Frontmatter** (YAML): 스킬 트리거링을 위한 `name`과 `description` 필드 포함
- **Body** (마크다운): 스킬 사용을 위한 지침과 안내

### 번들 리소스 (선택사항)

- **Scripts** (`scripts/`): 확정적 신뢰성을 위한 실행 가능한 코드
- **References** (`references/`): 필요에 따라 컨텍스트에 로드되는 문서
- **Assets** (`assets/`): 출력에 사용되는 파일들 (템플릿, 이미지, 보일러플레이트)

## 스킬 생성 과정

1. **구체적인 예제로 스킬 이해하기**
2. **재사용 가능한 스킬 콘텐츠 계획** (스크립트, 참조 자료, 자산)
3. **스킬 초기화** (디렉토리 구조 생성)
4. **스킬 편집** (리소스 구현 및 SKILL.md 작성)
5. **스킬 패키징** (배포 가능한 .skill 파일 생성)
6. **실제 사용을 기반으로 반복 개선**

### 1단계: 스킬 이해하기

스킬이 어떻게 사용될지에 대한 구체적인 예제를 명확히 이해합니다. 다음과 같은 질문을 해보세요:
- "이 스킬이 어떤 기능을 지원해야 하나요?"
- "이 스킬이 어떻게 사용될지 예제를 들어줄 수 있나요?"
- "사용자가 이 스킬을 트리거하려면 무엇이라고 말해야 하나요?"

### 2단계: 재사용 가능한 콘텐츠 계획

각 예제를 분석하여 도움이 될 스크립트, 참조 자료, 자산을 식별합니다:
- 반복적으로 다시 작성되는 코드를 위한 스크립트
- 스키마와 문서를 위한 참조 자료
- 템플릿과 보일러플레이트를 위한 자산

### 3단계: 스킬 초기화

스킬 디렉토리 구조를 생성합니다:

```bash
mkdir 스킬명
cd 스킬명
mkdir scripts references assets
```

### 4단계: 스킬 편집

#### SKILL.md 작성

**Frontmatter:**
```yaml
---
name: 스킬명
description: 스킬이 하는 일과 언제 사용해야 하는지 포함한 명확한 설명
---
```

**Body:** 스킬과 번들 리소스 사용을 위한 지침.

#### 점진적 공개 패턴

SKILL.md를 500줄 미만으로 유지하세요. 이 한계에 접근할 때 콘텐츠를 분할하세요:

- **참조가 있는 고수준 가이드**: 상세 가이드로 링크
- **도메인별 구성**: 도메인/프레임워크별로 파일 분리
- **조건부 세부사항**: 고급 기능으로의 링크가 있는 기본 콘텐츠

### 5단계: 스킬 패키징

적절한 디렉토리 구조를 가진 모든 파일을 포함하는 배포 가능한 .skill 파일 (확장자가 .skill인 zip 파일)을 생성합니다.

### 6단계: 반복 개선

1. 실제 작업에서 스킬 사용
2. 어려움이나 비효율성 발견
3. 필요한 업데이트 식별
4. 변경사항 구현 및 다시 테스트

## 모범 사례

- SKILL.md를 간결하고 집중적으로 유지
- 지침에 명령형/부정사형 사용
- 트리거 정보를 본문이 아닌 description에 포함
- SKILL.md와 references 간 중복 방지
- 스크립트가 올바르게 작동하는지 테스트
- 긴 참조 파일을 목차와 함께 구조화
